# План миграции на Alchemy + Viem (один контракт в день)

## Цель
Полностью перевести критически важные функции проекта с инфраструктуры Thirdweb на прямой RPC Alchemy через библиотеку Viem. Это снизит нагрузку, повысит стабильность и защитит проект от сбоев Thirdweb.

## ИНСТРУКЦИЯ ПО ПЕРЕЕЗДУ (ALCHEMY + VIEM)

### Шаг 1: Подготовка (ENV)
Убедитесь, что в `.env` прописан актуальный URL:
`VITE_ALCHEMY_GM_RPC_URL=https://base-mainnet.g.alchemy.com/v2/ВАШ_КЛЮЧ`

### Шаг 2: Чтение данных (Read)
Вместо хука `useReadContract` из Thirdweb используем `publicClient` из `@/lib/viem/client`:

```typescript
// Было (Thirdweb):
const { data } = useReadContract({ contract, method: "balanceOf", params: [address] });

// Стало (Viem + React Query):
const { data } = useQuery({
  queryKey: ['balance', address],
  queryFn: () => publicClient.readContract({
    address: CONTRACT_ADDRESS as `0x${string}`,
    abi: CONTRACT_ABI,
    functionName: 'balanceOf',
    args: [address], // Используем args вместо params!
  })
});
```

### Шаг 3: Транзакции (Write)
Вместо `prepareContractCall` и `useSendTransaction` используем прямой метод аккаунта:

```typescript
// Внутри мутации или функции:
const account = useActiveAccount();

const data = encodeFunctionData({
  abi: CONTRACT_ABI,
  functionName: 'yourFunction',
  args: [arg1, arg2],
});

const { transactionHash } = await account.sendTransaction({
  to: CONTRACT_ADDRESS as `0x${string}`,
  data,
  chainId: 8453, // ОБЯЗАТЕЛЬНО для Base
});

// Ожидание подтверждения через Alchemy:
await publicClient.waitForTransactionReceipt({ hash: transactionHash as `0x${string}` });
```

## Полезные советы
1. **Типизация:** Всегда приводите адреса к `as 0x${string}`.
2. **Аргументы:** В Viem поле называется `args`, а не `params`.
3. **Кэш:** После транзакции не забывайте делать `queryClient.invalidateQueries`.
4. **Alchemy Dashboard:** Проверяйте вкладку "Mempool" в Alchemy, чтобы видеть транзакции в реальном времени.
